## 修正内容

### 1. SQL-001 — IN句での大量データ外部テーブル参照（L13-L18）

**修正前:**
```sql
DELETE FROM orders
WHERE customer_id IN (
    SELECT customer_id
    FROM external_customers
);
```

**修正後:**
```sql
DELETE FROM orders o
WHERE EXISTS (
    SELECT 1
    FROM external_customers ec
    WHERE ec.customer_id = o.customer_id
);  -- Fixed: SQL-001
```

**修正理由:**
EXISTSは該当行が見つかった時点でスキャンを停止する（短絡評価）。IN句はサブクエリの結果セット全体を先に構築してからメインクエリと照合するため、外部テーブルのデータ量が多いほどパフォーマンスが劣化する。

### 2. SQL-001 — IN句での大量データ外部テーブル参照（L21-L27）

**修正前:**
```sql
SELECT order_id, order_date, total_amount
FROM orders
WHERE customer_id IN (
    SELECT customer_id
    FROM external_master_data
    WHERE status = 'active'
);
```

**修正後:**
```sql
SELECT order_id, order_date, total_amount
FROM orders o
WHERE EXISTS (
    SELECT 1
    FROM external_master_data emd
    WHERE emd.customer_id = o.customer_id
    AND   emd.status = 'active'
);  -- Fixed: SQL-001
```

**修正理由:**
EXISTS書き換えにより、外部テーブルの行数に依存しない効率的な結合処理が可能になる。

### 4. SQL-002 — SELECT * の使用（L41）

**修正前:**
```sql

```

```sql
WHERE  o.order_date >= '2026-01-01';
```

**修正後:**

### 6. SQL-003 — WHERE句なしのDELETE（L51）

**修正前:**
```sql
DELETE FROM tmp_batch_work;
```

**修正後:**
```sql
TRUNCATE TABLE tmp_batch_work;  -- Fixed: SQL-003
```

**修正理由:**
WHERE句なしのDELETEは全行を1行ずつ削除ログを書きながら処理するため極めて低速。TRUNCATEはページ単位で解放するため高速。ただしTRUNCATEはロールバック不可のため用途を確認すること。

### 8. SQL-004 — 暗黙の型変換によるインデックス無効化（L60-L61）

### 12. SQL-006 — NULLの等価比較（L78）

**修正前:**
```sql
SELECT * FROM orders WHERE discount = NULL;
```

**修正後:**
```sql
SELECT * FROM orders WHERE discount IS NULL;  -- Fixed: SQL-006
```

**修正理由:**
SQLの三値論理では `NULL = NULL` は TRUE ではなく UNKNOWN を返す。NULL判定には必ず `IS NULL` を使用すること。

---

### 13. SQL-006 — NULLの等価比較（L80-L82）

**修正前:**

**修正後:**

### 14. SQL-007 — 相関サブクエリのSELECT句使用（L88-L95）

**修正前:**
```sql
SELECT
    e.employee_id,
    e.name,
    (SELECT d.department_name
     FROM   departments d
     WHERE  d.department_id = e.department_id) AS dept_name,
    (SELECT COUNT(*)
     FROM   order_items oi
     WHERE  oi.assigned_to = e.employee_id) AS item_count
FROM employees e
WHERE e.is_active = 1;
```

**修正後:**
```sql
SELECT
    e.employee_id,
    e.name,
    d.department_name AS dept_name,
    COUNT(oi.item_id) AS item_count
FROM   employees e
LEFT JOIN departments d
    ON d.department_id = e.department_id
LEFT JOIN order_items oi
    ON oi.assigned_to = e.employee_id
WHERE  e.is_active = 1
GROUP BY e.employee_id, e.name, d.department_name;  -- Fixed: SQL-007
```

**修正理由:**
JOINに書き換えることで、1回のクエリで結合処理が完了する。相関サブクエリはメインクエリの各行に対して繰り返し実行されるため、行数増加に比例して処理時間が増大する。

## 注意事項

- SQL-003（WHERE句なしDELETE → TRUNCATE）: TRUNCATEはロールバック不可です。意図的な全行削除である場合のみ適用してください。条件付き削除が本来の意図であればWHERE句を追加してください。
- SQL-002（SELECT *）: 修正後のカラム名はテーブル定義に基づく推定です。実際のカラム名を確認の上、適用してください。
- 全修正は **動作の同一性を保証する** 方針で行っています。修正前後でクエリの結果セットまたは影響行が同一であることを確認してください。
