## 詳細

### 1. SQL-001 — IN句での大量データ外部テーブル参照（high）

**該当コード（L13-L18）:**
```sql
DELETE FROM orders
WHERE customer_id IN (
    SELECT customer_id
    FROM external_customers
);
```

**問題点:**
IN句のサブクエリで外部テーブルを参照している場合、データ量が増加するにつれてフルテーブルスキャンが発生し、処理時間が線形以上に増大する。EXISTSは該当行が見つかった時点でスキャンを停止する（短絡評価）が、IN句はサブクエリの結果セット全体を先に構築する。

**改善案:**
```sql
DELETE FROM orders o
WHERE EXISTS (
    SELECT 1
    FROM external_customers ec
    WHERE ec.customer_id = o.customer_id
);
```

### 2. SQL-002 — SELECT * の使用（medium）

**該当コード（L41）:**
```sql
SELECT * FROM employees WHERE department_id = 10;
```

**問題点:**
SELECT * はテーブルの全カラムを取得するため、不要なデータの転送・メモリ使用が発生する。カラム追加時に意図しないデータが返却されるリスクもある。

**改善案:**
```sql
SELECT employee_id, name, email
FROM employees
WHERE department_id = 10;
```

---

**該当コード（L43-L45）:**

SELECT * FROM orders o
JOIN   order_items oi ON oi.order_id = o.order_id
WHERE  o.order_date >= '2026-01-01';

**改善案:**
```sql
SELECT o.order_id, o.order_date, o.total_amount,

```

### 7. SQL-007 — 相関サブクエリのSELECT句使用（medium）

**該当コード（L88-L95）:**
```sql
SELECT
    e.employee_id,
    e.name,
    (SELECT d.department_name
     FROM   departments d
     WHERE  d.department_id = e.department_id) AS dept_name,
    (SELECT COUNT(*)
     FROM   order_items oi
     WHERE  oi.assigned_to = e.employee_id) AS item_count
FROM employees e
WHERE e.is_active = 1;
```

**問題点:**
SELECT句に相関サブクエリを2つ記述しており、メインクエリの行数分だけ各サブクエリが実行されるN+1問題が発生する。

**改善案:**
```sql
SELECT
    e.employee_id,
    e.name,
    d.department_name AS dept_name,
    COUNT(oi.item_id) AS item_count
FROM   employees e
LEFT JOIN departments d
    ON d.department_id = e.department_id
LEFT JOIN order_items oi
    ON oi.assigned_to = e.employee_id
WHERE  e.is_active = 1
GROUP BY e.employee_id, e.name, d.department_name;
```
